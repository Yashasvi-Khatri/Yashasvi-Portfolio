Deep Dive

CarbonCompanion
— featured case study

★ DRDO-Mentored Project
AI-Powered Carbon
Footprint Management

An end-to-end web application that uses machine learning to predict carbon emissions based on user behavior, transportation patterns, and energy consumption — then serves personalized reduction strategies. Built to address the gap between static calculators and truly intelligent environmental tools.

Mentor
Phuldeep Kumar, Scientist, DRDO
Institution
VIPS-TC · AI & Data Science
Architecture
MERN + Python ML Pipeline
AI Stack
TensorFlow.js · Scikit-learn · Regression Models
Architecture Decisions
  • React frontend decoupled from Node.js backend via REST API — enables independent scaling
  • TensorFlow.js runs lightweight inference in-browser; Python/Scikit-learn for heavier training pipeline
  • MongoDB chosen over SQL for flexible, user-specific activity schemas
  • JWT-based stateless auth — no session storage overhead
  • Chart.js for real-time visualizations without heavy bundle cost
Key Features
  • ML regression models predict future emissions from historical behavior
  • Personalized reduction roadmap generated per user profile
  • Environmental data API integration for regional emission factors
  • Gamified goal tracking to sustain user engagement
  • Comparative analysis against regional and global averages
  • Admin dashboard for aggregate insights
Engineering Challenges
  • Normalizing inconsistent input data across transportation modes and energy types
  • Making ML predictions interpretable for non-technical users
  • Balancing prediction accuracy with model size for browser-side inference
  • Designing a schema that handles diverse lifestyle activity types
  • Ensuring recommendation relevance without overfitting to sparse user data

About

Why I build
software

I'm a Full Stack Engineer and AI & Data Science undergraduate at VIPS-TC, GGSIPU (graduating 2027). I got into software because I was obsessed with the idea that code — more than almost any other medium — lets one person's idea become something millions of people use.

That obsession led me to DRDO as an R&D intern, where I worked in engineering environments where the cost of a bug isn't a bad Slack message — it's a mission failure. That experience permanently changed how I think about code quality, documentation, and reliability.

My interest in AI-powered products isn't trend-chasing. I published research on applying ML to environmental systems (CarbonCompanion) and on how AI transforms music education. I believe the most interesting engineering problems right now live at the intersection of solid software architecture and applied machine learning.

I gravitate toward problems where the user experience and the technical architecture have to be solved together — not independently. Building CarbonCompanion taught me that even the best ML model is useless if the interface makes its outputs incomprehensible.

Engineering Philosophy

"Good software is honest software. It does what it says, scales when it needs to, and fails loudly when it breaks. The rest is just aesthetics."
